# Huashu Design Direction — 设计方向顾问模式

## 触发条件

当用户**没有**指定任何风格描述、品牌色或模板时，Strategist 在 Confirmation d 中激活顾问模式。

## 顾问模式工作流

### 第 1 步：内容特征分析

分析源文档，提取 3 个特征：
1. **行业/领域**：科技、金融、医疗、教育、文化...
2. **内容密度**：图文为主、数据密集、文字主导、混合
3. **情感基调**：严肃、活泼、专业、创新、温暖...

### 第 2 步：生成 3 个差异化方向

从 40 种风格库中选择 3 个**差异足够大**的方向：

| 维度 | 方向 A（保守） | 方向 B（中等） | 方向 C（大胆） |
|------|---------------|---------------|---------------|
| 定位 | 行业惯例 | 略有新意 | 突破常规 |
| 风险 | 零风险 | 可控风险 | 需要说服 |
| 适用 | 保守行业（金融/政务） | 一般商业场景 | 创意/科技行业 |

### 第 3 步：每个方向的呈现格式

```
[方案 A] <风格名称> — <一句话描述>
  视觉：字体搭配 / 色彩倾向 / 布局特征
  适合：适用场景和受众
  参考：真实世界类比（公司/出版物/活动）
```

### 第 4 步：用户选择后的下游影响

用户选择方向后，Confirmation e-h 自动对齐：

| 确认项 | 对齐内容 |
|--------|---------|
| e. 配色 | 该风格的推荐色板（HEX） |
| f. 图标 | 该风格的图标风格（线条/填充/双色） |
| g. 排版 | 该风格的字体搭配建议 |
| h. 图片 | 该风格的图片渲染 + 调色板 |

## 品位锚点（Taste Anchors）

品位锚点是防止 AI slop 的正面表述——不是"不要什么"，而是"要什么"。

| 锚点类型 | 示例 | 效果 |
|---------|------|------|
| 字体 | "衬线 display + 无衬线 body" | editorial 感 |
| 色彩 | "单一暖色贯穿全篇" | 克制、聚焦 |
| 密度 | "信息密度分型：封面 breathing，正文 dense" | 节奏感 |
| 材质 | "纸张纹理 8% 透明度" |  tactile 感 |

## 反 AI Slop 清单

以下模式在融合技能中**始终禁止**：

| 模式 | 为什么不好 | 替代方案 |
|------|-----------|---------|
| 紫色渐变背景 | AI 默认审美 | 实色背景 + 单一强调色 |
| Emoji 图标 | 不专业、风格不统一 | SVG 图标库（chunk/tabler/phosphor） |
| 圆角卡片 + 左边框 | 千篇一律的"AI 感" | 无边框分割 / 网格 / 非对称布局 |
| SVG 绘制的人物剪影 | 廉价感、过时 | 照片 / 矢量插画 / 抽象图形 |
| 每页密度一致 | 没有节奏 | breathing/dense/anchor 交替 |
